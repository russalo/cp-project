# GEMINI.md

This file provides Gemini-facing repository guidance. It mirrors the shared policy in
`docs/reference/ai-guidance.md`.

## Documentation Pipeline

- `INBOX.md`: raw capture only
- `VISION.md`: future-scope ideas not yet sequenced into active work
- `DECISIONS_INBOX.md`: pending design decisions awaiting approval
- `MILESTONES_INBOX.md`: draft sequencing awaiting approval
- `CHARTER.md`: active scope and current explicit rules
- `DECISIONS.md`: accepted canonical decision log
- `MILESTONES.md`: formal roadmap and active sequencing
- `docs/archive/`: preserved history and superseded notes

## Working Rules

- Use `CHARTER.md`, `DECISIONS.md`, and `MILESTONES.md` as the active source of truth.
- Do not write raw brainstorming directly into canonical docs.
- Route unapproved ideas to the inbox or review-layer docs first.
- Preserve historical notes by moving them into `docs/archive/` instead of deleting them.
- Review `KNOWLEDGE-PIPELINE.md` whenever the documentation structure changes.
- Keep behavior aligned with `docs/reference/ai-guidance.md`.
